2. Identity & Fraud Controls

  • Onboarding: Mandatory WhatsApp OTP verification to ensure the "Phone-Identity" link.
  • Proximity Rule: System flags any "Release Code" entry attempted from a GPS location > 5km from the original "Job Started" coordinate (prevents remote theft).
  • Tax ID Validation: Per local tax administration mandates, all Providers must link a valid Tax ID via our verification partner before their first payout.
  • Banning: Permanent hardware-ID ban for any account associated with a "Stolen Code" or "Fraudulent Refund" incident.

3. Incident & Breach Response Plan

In the event of a suspected data leak or unauthorized access:

  1. Detection: Real-time monitoring of failed login attempts and unauthorized API calls via server logs.
  2. Containment: Immediate rotation of API keys and temporary lockdown of the affected user Node.
  3. Notification (72-Hour Rule): We will notify the local Data Protection Authority and affected users within 72 hours of a confirmed breach.
  4. Recovery: Restore from the last "Clean" encrypted backup; patch the vulnerability.

4. Consent & Privacy UI

  • Explicit Consent: A mandatory "I agree to the LocaLe Privacy Policy" checkbox on the signup modal.
  • Age Gate: Users must confirm they are 18+ to engage in financial escrow.
  • Data Request Tool: A simple "Export My Data" button in the profile settings to comply with the Right to Portability.
